2. Abstract

Existing world models for autonomous driving struggle with long-horizon generation and generalization to challenging scenarios. In this work, we develop a model using simple design choices, and without additional supervision or sensors, such as maps, depth, or multiple cameras. We show that our model yields state-of-the-art performance, despite having only 469M parameters and being trained on 280h of video data. It particularly stands out in difficult scenarios like turning maneuvers and urban traffic. We test whether discrete token models possibly have advantages over continuous models based on flow matching. To this end, we set up a hybrid tokenizer that is compatible with both approaches and allows for a side-by-side comparison. Our study concludes in favor of the continuous autoregressive model, which is less brittle on individual design choices and more powerful than the model built on discrete tokens.

4. Problem and Motivation

Storyline

Driving world models aim to simulate future visual states for planning and representation learning. Recent diffusion-based models (Vista, GEM) produce high-quality short-horizon videos but fail systematically on turns and rollouts beyond ~5 seconds, exhibiting premature stopping and lateral jitter. Existing metrics (FVD, JEDi) use general-purpose encoders insensitive to ego-motion quality. The paper questions whether continuous latent spaces are the root cause and tests this via a controlled comparison using a novel hybrid tokenizer. The resulting continuous model (Orbis) achieves SOTA performance while being data-efficient.

5. Methodology

Overall Framework

[Input Frame I] -+-> [Semantic Encoder Es + DINOv2 distillation] -> [xs] -> [Qs] -> [qs]
                 +-> [Detail Encoder Ed]                           -> [xd] -> [Qd] -> [qd]
                      (50% training: bypass VQ -> continuous tokens)

[Context Frames x_{0:N-1}] + [Target x_N^tau (noise/mask)]
  -> [STDiT World Model]
     (Spatial Attn + Temporal Attn + AdaLN conditioning)
  -> [M iterative denoising (FM) or unmasking (MGM) steps]
  -> [Generated x_N^0] -> [Decoder] -> [Reconstructed Frame]

Rollout: append x_N^0 to context, drop x_0, repeat
[Orbis-MG only] -> [Temporal Refinement 30M U-Net] -> [Smoothed Frame]

Hybrid Image Tokenizer

Factorized VQ-VAE with separate semantic (ViT, DINOv2-distilled) and detail encoders. Each branch has its own codebook (2x4096 vocab). Fine-tuned with 50% VQ bypass probability to support both discrete and continuous modes. EQ-VAE implicit regularization applied. 234M parameters; 16x spatial compression.

Key insight: Token factorization is critical for discrete models (FVD: 1214 → 533); continuous models are robust to this choice (248 → 246).

Flow Matching World Model (Orbis)

Next-frame prediction via ODE integration. Forward: $x_N^\tau = (1-\tau)x_N + \tau\epsilon$. Model predicts velocity to denoising direction conditioned on context frames.

$$L = \mathbb{E}_{\tau,\epsilon}\|v(x_N^\tau; x_{0:N-1}) - (\epsilon - x_N)\|^2$$

Inference: 30 ODE steps. Context dropout (50%) + context noise augmentation (50%). 469M params; 72x A100, 5 days training.

Masked Generative World Model (Orbis-MG)

Discrete next-frame prediction via iterative unmasking (MaskGIT objective). Target frame fully masked at inference; confidence-based unmasking over multiple iterations. Token-similarity regularization from VQ codebooks reintroduces pairwise structure. Context augmentation: 10% frame replacement + 10% token masking for robustness.

$$L_{CE} = \mathbb{E}_M\left[-\sum_i \log p_\theta\left(q_N^{(i)}\,|\,q_{0:N-1}, q_N^M\right)\right]$$

Known shortcoming: Discrete model selects the same token as last context frame ~45% of the time (vs. 29% for real frames), reflecting content-copying bias in discrete space that is mitigated but not eliminated.

7. Phyra's Judgment

Supported Contributions

  • Shortcomings of SOTA models on turning and long-horizon, supported by Figure 2, Table 2, Figure 4
  • Continuous flow matching outperforms discrete MaskGIT on fair common ground, supported by Tables 3, 4, 6 across datasets and architectures
  • 469M Orbis sets SOTA on long-horizon rollouts with 280h training data, supported by Table 3

Overclaimed or Preliminary

  • "Hybrid" tokenizer: continuous model performs similarly with simpler tokenizers (Table 4 ablation), reducing the practical impact of hybrid compatibility
  • Ego-motion controllability: no comparison to other controllable models; ADE only on 400 nuPlan sequences; labeled "preliminary" by authors

Fundamental Limitations

  • Sliding-window context (5 frames): Discards early context permanently; no memory of states beyond ~1 second, limiting complex trajectory recall.
  • Raw-video-only design: Bounds representation to daytime front-camera; multi-view and adverse weather generalization is structurally absent, not just a data gap.
  • Discrete copying artifact: 45% same-token prediction (vs. 29% real) in Orbis-MG is mitigated but not resolved by regularization, indicating fundamental expressivity constraint in discrete space.
  • SVD failure cause unresolved: Orbis avoids SVD pretraining but does not identify the mechanism that causes SVD-based models to fail at long horizons.

8. Open Questions

  • Why do SVD-based diffusion models fail at long-horizon driving prediction despite strong short-horizon quality? Is the failure in latent space structure, training objective, or temporal attention design?
  • Does the flow matching advantage over MaskGIT hold when scaling both to larger parameter counts and longer context lengths?
  • Can the semantic branch be replaced with stronger video self-supervised representations (e.g., V-JEPA-2 [2]) to improve temporal consistency?
  • Does the improved state representation in Orbis translate to downstream planning gains on closed-loop benchmarks (e.g., nuPlan)?
