Core Idea

PIWM extends DreamerV3's RSSM into an individual-level branched architecture where each vehicle (ego, VDI, VPI) maintains its own latent state. Inter-vehicle relations are captured via self-attention over deterministic states. The decoder predicts future trajectories instead of reconstructing current observations, encoding intention into latent states. An actor-critic policy then acts on intention-aware states fused by cross-attention.

Highlights

  1. Individual-level branched RSSM reduces feature entanglement; IWM ablation shows +10% success rate vs. DreamerV3 on small-scale (Table II).
  2. Replacing reconstruction with trajectory prediction encodes intention into latent states; visualized in Fig. 9 with decoded future trajectories.
  3. +18.81% success rate over DreamerV3 on 156-scenario large-scale unseen evaluation (74.79% vs. 55.98%, Table I).
  4. Cross-attention provides explicit ego-centric fusion, distinguished from the self-attention used in the transition model.

Weaknesses

  1. Background agents are log-replayed (non-reactive); interactive decision-making is evaluated against fixed trajectories.
  2. Only one map (DR_USA_Intersection_EP0) from INTERACTION dataset; no cross-map generalization.
  3. VPI decoder absent by design; rationale is heuristic, not ablated.
  4. VDI prediction ADE is high; map information is explicitly excluded.
  5. No wall-clock latency or training time reported.

3. Problem and Motivation

Storyline

Reactive urban driving requires modeling other agents' unknown intentions. Model-free RL lacks sample efficiency and long-term planning. Model-based RL (Dreamer) addresses sample efficiency via world model imagination, but prior Dreamer-AD methods use scene-level representations that mingle vehicle features and reconstruction-based learning that fails to capture long-term intentions. PIWM targets both gaps simultaneously.

Pain Points

4. Method: PIWM Architecture

Data Flow

Observation x_t  (ego + VDI{Nd=5} + VPI{Np=5}  historical 2s trajectories @ 10Hz)
     |
     +--[Shared Trajectory Encoder Enc_phi]--> common features
          |
          +--[Ego Branch Enc_theta1]  --> e^ego_t
          +--[VDI Branch Enc_theta2]  --> e^VDI_t
          +--[VPI Branch Enc_theta3]  --> e^VPI_t
          |
     [Branched GRU cells]  -->  h^{ego, VDI, VPI}_t
          |
     [Self-Attention over h_t]  -->  self_att_t
          |
     [Stochastic sampling]
          prior: z-bar_t ~ p(z-bar_t | h_t, self_att_t)
       posterior: z_t    ~ q(z_t    | h_t, self_att_t, e_t)
          |
       s_t = {h_t, z_t}
          |
     +----+----+
     |         |
[Traj Dec]  [Cross-Att(s^ego, s^VDI)] --> cross_att_t
y-hat_t          |
(ego + VDI)  +---+---+
             |       |
        [Reward]  [Continuation]  [Actor pi] --> a_t  [Critic u] --> v_t

Branched Encoding

Vehicles split into ego, VDI (closest Nd=5), and VPI (next Np=5) by distance. Each group passes through a shared trajectory encoder (Enc_phi) then a group-specific MLP branch. Dynamic membership: newly admitted vehicles initialize latent states to zero.

$$e^{ego}_t = \text{Enc}_{\theta_1}(\text{Enc}_\phi(x^{ego}_t))$$ $$e^{VDI}_t = \text{Enc}_{\theta_2}(\text{Enc}_\phi(x^{VDI}_t))$$ $$e^{VPI}_t = \text{Enc}_{\theta_3}(\text{Enc}_\phi(x^{VPI}_t))$$

Interaction-Enhanced Transition

Branched GRUs update deterministic states. Self-attention over concatenated h_t produces inter-vehicle relation features self_att_t appended during stochastic state sampling.

$$\bar{z}_t \sim p(\bar{z}_t \mid h_t, \text{self\_att}_t)$$ $$z_t \sim q(z_t \mid h_t, \text{self\_att}_t, e_t)$$

World Model Loss

$$\mathcal{L}_{PIWM} = \mathbb{E}\sum_{t=1}^{T}\Big[ \underbrace{-\ln p(y^{ego}_t|s^{ego}_t) - \ln p(y^{VDI}_t|s^{VDI}_t)}_{\text{prediction log loss}} \underbrace{- \ln p(r_t|\cdot) - \ln p(c_t|\cdot)}_{\text{reward + continuation}} + \underbrace{\beta\,\text{KL}[\text{post}\|\text{prior}]_{\{ego,VDI,VPI\}}}_{\text{KL, }\beta=0.5} \Big]$$

Behavior Model

Actor-critic follows DreamerV3. States fused via cross-attention (ego-centric): query = ego state, keys/values = ego + VDI states. Actor outputs discrete onehot distribution over 4 speed targets {0, 3, 6, 9} m/s. Critic uses twohot softmax distribution (distributional RL, 41 buckets from −20 to +20).

$$\text{at} \sim \pi_\omega(a_t \mid s^{ego}_t,\, \text{cross\_att}_t) \qquad v_t \sim u_\psi(s^{ego}_t,\, \text{cross\_att}_t)$$

Individual modeling alone (IWM) gives a moderate gain in small-scale; the gain shrinks in large-scale, suggesting reconstruction-based learning does not scale to diverse scenarios. Adding trajectory prediction (PIWM) produces the dominant gain in both settings.

Prediction Horizon Ablation (Fig. 8)

Horizons tested: H+ ∈ {10, 15, 20, 25, 30, 35, 40} steps (0.1s each). H+ = 20 (2s) achieves best success-rate / ADE trade-off. Shorter horizons fail to encode long-term intent; longer horizons accumulate prediction error that degrades the representation quality and decision performance.

Red Flag: Self-attention and trajectory prediction are introduced together in PIWM; no ablation isolates the contribution of self-attention alone from the contribution of the prediction objective within the PIWM configuration.
Red Flag: All experiments use one map (DR_USA_Intersection_EP0); generalizability claims are not supported beyond this single-map, log-replay, non-reactive traffic setting.

Fundamental Limitations

  1. Non-reactive simulation ceiling: Log-replay background agents reduce interactive decision-making to timing-based gap acceptance, not genuine multi-agent interaction.
  2. No map conditioning: Excluding road geometry from trajectory decoders limits prediction quality; VDI ADE is high by the authors' own acknowledgment.
  3. Online data collection bias: VDI training data collected from ego's perspective during online RL -- inherently limited and ego-centric in distribution.
  4. Hard VDI/VPI split: Fixed Nd=Np=5 with a distance threshold cannot adapt to scenarios with variable agent count or spatial distributions.

Open Questions

  • Would learned attention-based vehicle importance weighting outperform the hard distance threshold?
  • How does prediction ADE correlate with downstream success rate when map information is included?
  • Does the individual-level representation still confer advantages against reactive background agents?
  • What is PIWM's inference latency relative to DreamerV3?

Improvement Directions (by feasibility)

  1. Include map conditioning in trajectory decoder -- OSM map files already available in I-SIM; directly addresses high VDI ADE.
  2. Replace log-replay with reactive agents -- I-SIM supports interactive agents; would strengthen interactive decision-making claim.
  3. Learned vehicle relevance weighting -- Replace hard distance-based VDI/VPI split with soft attention-based relevance score.
  4. Offline pre-training of PIWM -- Pre-train on full INTERACTION dataset before online RL fine-tuning to address VDI data limitation.
