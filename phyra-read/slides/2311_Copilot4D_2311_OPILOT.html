Copilot4D: Learning Unsupervised World Models for Autonomous Driving via Discrete Diffusion

Lunjun Zhang, Yuwen Xiong, Ze Yang, Sergio Casas, Rui Hu, Raquel Urtasun

World Model Discrete Diffusion Point Cloud Forecasting VQVAE Autonomous Driving

Core Idea

Copilot4D demonstrates that tokenizing LiDAR point clouds into discrete BEV tokens and then predicting future tokens via discrete diffusion yields a scalable unsupervised world model for autonomous driving.

Two bottlenecks addressed:

  • Observation space complexity: 3D point clouds are unstructured and high-dimensional. Solved by VQVAE tokenization into a 128x128 BEV token grid.
  • Scalable generative model: Prior methods use task-specific architectures. Solved by discrete diffusion (modified MaskGIT) with a spatio-temporal Transformer.

Architecture Overview

LiDAR Point Cloud | v PointNet Voxel Encoder --> BEV Pillar Pooling | v Swin Transformer Encoder --> Vector Quantization | v Discrete BEV Tokens (128x128, vocab=1024) | v + ego-vehicle poses (actions) Spatio-Temporal Transformer (U-Net, 3 resolution levels) | v Discrete Diffusion (K=10 steps) Predicted BEV Tokens | v VQVAE Decoder (NFG + Depth Rendering) | v Predicted Point Cloud

The tokenizer is 13M parameters; the world model is 39M parameters (p.14). Both are smaller than a ResNet-34.

VQVAE-like Point Cloud Tokenizer

Encodes LiDAR observations into discrete BEV tokens and reconstructs point clouds via two decoder branches.

ComponentDetail
Encoder backbonePointNet voxel encoding + Swin Transformer (4+6 layers)
Token grid128 x 128 spatial, codebook size 1024
Decoder branch 1Neural Feature Grid (NFG): 3D implicit representation, bilinear interpolation, differentiable depth rendering
Decoder branch 2Coarse reconstruction: binary voxel prediction for spatial skipping at inference
Anti-collapseK-Means clustering on memory bank (10x codebook size), dead code re-init at 3% threshold (p.17)

Differentiable depth rendering is the key novelty over prior tokenizers like UltraLiDAR, which only predict binary voxel occupancy and cannot recover fine-grained geometry (Figure 9, p.17).

Discrete Diffusion (Modified MaskGIT)

The paper establishes a theoretical link between MaskGIT and absorbing diffusion models via the Campbell et al. (2022) parameterization (Eq. 6, p.5).

Two modifications turn MaskGIT into a proper discrete diffusion model:

  • Uniform noise injection: After masking, inject $\eta$% (default 20%) uniform noise into remaining tokens during training
  • Token resampling at inference: Already-decoded tokens can be re-evaluated and revised at each diffusion step

These changes reduce NuScenes 3s Chamfer by 29% compared to vanilla MaskGIT (Table 4, p.9).

Classifier-Free Diffusion Guidance (CFG)

Adapted to discrete tokens via logit-space interpolation:

$$\text{logits}_{cfg} = \text{logits}(\hat{x}_0 | x_{k+1}, c) + w \cdot \big(\text{logits}(\hat{x}_0 | x_{k+1}, c) - \text{logits}(\hat{x}_0 | x_{k+1})\big)$$

Implemented efficiently: extend sequence length by 1, use causal mask for conditional and identity mask for unconditional (Figure 10, p.20).

Strengths and Weaknesses

Strengths

  • First to validate the "tokenize + discrete diffusion" paradigm for 3D world models in driving, with 65-75% Chamfer reduction
  • Principled theoretical connection between MaskGIT and absorbing diffusion (Eq. 6)
  • Lightweight model (52M total params) achieving state-of-the-art results

Weaknesses

  • No downstream task evaluation (planning, RL, detection); the world model's utility is demonstrated only via forecasting metrics
  • Missing ablation on the 50/40/10 training objective mixture
  • BEV representation loses height information; fixed voxel resolution

Fundamental Limitations

  • BEV pooling discards vertical structure; cannot distinguish objects at different heights
  • Actions are ego-vehicle SE(3) poses only; other agents' behavior is not modeled
  • Autoregressive temporal rollout accumulates errors over long horizons

Open Questions

  • How does the model perform when ego-poses are predicted rather than given?
  • Can the model generate diverse multi-futures? (qualitative examples shown, no quantitative diversity metric)
  • Would scaling the Transformer and codebook yield further gains?
