Problem and Motivation

Reconstructing dynamic 3D street scenes from images is critical for autonomous driving simulation. Existing approaches face two bottlenecks:

  • NeRF-based methods (NSG, MARS, EmerNeRF) produce reasonable quality but require hours of training and render below 1 FPS (Table 1: EmerNeRF 0.21 FPS, MARS 0.68 FPS, p.10).
  • 3D Gaussian Splatting achieves real-time rendering for static scenes but has no mechanism for modeling dynamic moving objects (p.3).

Street Gaussians bridges this gap by extending 3D GS with an explicit compositional representation for dynamic urban scenes.

Method Overview

The scene is decomposed into separate 3D Gaussian point clouds:

  • Background model: Static scene elements in world coordinates, initialized from aggregated LiDAR + SfM point clouds.
  • Object models (x N): One per tracked vehicle, in local coordinates, with optimizable tracked poses for local-to-world transformation.

At render time, all point clouds are transformed to world coordinates, concatenated, and rendered via point-based alpha-blending. A sky cubemap handles distant regions.

Core Technical Components

1. 4D Spherical Harmonics (4D SH)

Standard SH coefficients cannot model time-varying vehicle appearance. Each SH coefficient $z_{m,l}$ is replaced by Fourier transform coefficients $f \in \mathbb{R}^k$, recovered at time $t$ via:

$$z_{m,l} = \sum_{i=0}^{k-1} f_i \cos\!\left(\frac{i\pi}{N_t}\,t\right)$$

This encodes lighting and appearance changes over time without per-frame storage (Eq. 3, p.6).

2. Tracked Pose Optimization

Off-the-shelf tracker poses are noisy. Learnable residuals $\Delta R_t$ (yaw) and $\Delta T_t$ (3D vector) refine each frame's pose:

$$R'_t = R_t \,\Delta R_t, \quad T'_t = T_t + \Delta T_t$$

This yields PSNR* improvement of 1.99 dB on moving objects (Table 3, p.11).

3. LiDAR Point Cloud Initialization

Aggregated LiDAR provides better initialization than SfM for urban scenes with under-observed and textureless regions. Objects with fewer than 2K LiDAR points use 8K random samples inside the bounding box (p.6-7).

Rendering Pipeline

[Background 3D GS]  +  [Object 3D GS x N, transformed via R_t, T_t]
                    |
             [Concatenate all points]
                    |
        [Project to 2D: mu' = KW*mu, Sigma' = JW*Sigma*W^T*J^T]
                    |
          [Alpha-blending: C = sum(c_i * alpha_i * prod(1 - alpha_j))]
                    |
        [Blend with sky cubemap: C_final = C_g + (1 - O_g) * C_sky]

No ray-marching required. Direct point projection enables real-time performance (Eq. 4-5, p.7-8).

Training Objective

$$\mathcal{L} = \mathcal{L}_{\text{color}} + \lambda_1 \mathcal{L}_{\text{depth}} + \lambda_2 \mathcal{L}_{\text{sky}} + \lambda_3 \mathcal{L}_{\text{sem}} + \lambda_4 \mathcal{L}_{\text{reg}}$$
TermDescriptionWeight
$\mathcal{L}_{\text{color}}$L1 + D-SSIM reconstruction loss$\lambda_{\text{SSIM}}=0.2$
$\mathcal{L}_{\text{depth}}$L1 depth vs LiDAR (95th percentile)$\lambda_1=0.01$
$\mathcal{L}_{\text{sky}}$Binary CE for sky mask$\lambda_2=0.05$
$\mathcal{L}_{\text{sem}}$Softmax CE for semantic logits$\lambda_3=0.1$
$\mathcal{L}_{\text{reg}}$Regularization (floater removal)$\lambda_4$ (see supp.)

Training: 30K iterations with Adam optimizer on single RTX 4090 GPU (p.9).

Applications

Scene Editing (Fig. 7, p.13)

The explicit decomposition enables vehicle rotation, translation, and swapping without re-training. Each object is an independent Gaussian point cloud that can be freely manipulated.

Object Decomposition (Fig. 8, p.13)

Foreground vehicles are cleanly separated from background. NSG fails to disentangle objects; MARS produces floaters. Street Gaussians produces high-fidelity decomposed results.

Semantic Segmentation (Table 4, p.13)

Rendered semantic maps achieve 58.81 mIoU on KITTI, surpassing Video K-Net predictions on both ground-truth images (57.94) and rendered images (53.81).

Limitations and Open Questions

  • Tracker dependency: Requires pre-computed 3D bounding boxes; untracked objects cause artifacts. No failure case analysis is provided.
  • Vehicles only: Pedestrians, cyclists, and other dynamic agents are not modeled as foreground objects.
  • Rigid body assumption: Cannot handle articulated or deformable objects (e.g., trucks with trailers).

Improvement Directions

  • End-to-end joint tracking and Gaussian optimization to remove external detector dependency.
  • Non-rigid extensions using part-based Gaussians or learned deformation fields.
  • Replace fixed Fourier basis with adaptive temporal modeling for non-periodic appearance changes.
