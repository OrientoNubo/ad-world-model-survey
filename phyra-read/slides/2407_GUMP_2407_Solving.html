Autonomous Driving World Model Traffic Simulation Motion Planning Generative Model

Core Idea

GUMP is a unified generative model that learns the dynamics of driving scenes through autoregressive next-token prediction over traffic agent states. A single trained model serves as the foundation for four downstream tasks:

Scene Generation

Creates traffic scenarios from scratch conditioned on maps and text prompts. Uses full-AR mode to sequentially generate each agent's initial state.

Reactive Simulation

Extrapolates future states from historical data in closed-loop. Uses partial-AR mode with intra-frame NAR conversion for speed.

Interactive Planning

Evaluates candidate driving policies by generating rollouts and scoring accumulated rewards. Replaces scripted simulators.

Online RL Training

Provides a data-driven world model as environment for reinforcement learning, replacing rule-based transition functions.

Key Design: Key-Value Pair Tokenizer

Unlike prior work (Trajeglish, MotionLM) that tokenizes action space, GUMP directly quantizes state space. Each agent is represented by two tokens per timestep:

Token TypeContentEmbedding
Key (control)Agent unique ID + object category$e^i_{\text{key}} = \text{PE}(i) + \sum_{s \in \{\text{id}, \text{class}\}} \text{Embed}(s)$
Value (state)Position, heading, velocity, size$e^i_{\text{value}} = \text{PE}(i) + \sum_{s} (\text{Embed}(s) + \text{sinusoidal}(s))$

Advantages of state-space tokenization (p. 7):

  • Each token has fixed semantics independent of generation history (no compounding errors)
  • Enables flexible agent addition/removal during generation
  • Supports both full-AR and partial-AR decoding within the same model

Architecture Overview

Four main components process static map and dynamic agent information (Fig. 2, p. 6):

ComponentFunctionDetail
2D CNN EncoderEncodes rasterized static mapResNet-style; input: lanes, routes, obstacles as raster image
KV TokenizerDiscretizes dynamic agent statesKey-value pairs with learned + sinusoidal embeddings
Multimodal Causal Transformer (MCT)Autoregressive next-token predictionGPT-2 backbone + Gated Cross Attention (Flamingo-style) for static-dynamic fusion
GRU Token DecoderDecodes latent to state trajectoriesStacked GRU layers; decodes: size, velocity, then position/heading per timestep

Three model sizes: GUMP-small (55M), GUMP-base (184M), GUMP-medium (523M).

Efficiency: Partial-AR and Prediction Chunking

Intra-frame NAR Conversion

Full-AR mode predicts agents sequentially within each frame. Partial-AR mode predicts all agents in parallel using the GRU decoder, then optionally refines with one additional forward pass (Fig. 4, p. 9).

  • No custom attention masks needed
  • Compatible with FlashAttention [12, 13]
  • Comparable quality to full-AR (Appendix E.1)

Prediction Chunking + Temporal Aggregation

Each AR step predicts a chunk of future timesteps. Overlapping predictions are aggregated with exponential decay:

$$\hat{s}_t = \sum_{i=0}^{T} \gamma^i \cdot s^i_{t-i}$$

Optimal decay rate: $\gamma = 0.5$ (Appendix E.3, p. 33). This reduces compounding errors and improves simulation realism metrics.

Strengths and Weaknesses

Strengths

  • Single model serves four distinct downstream tasks; first unified demonstration in motion planning (p. 3)
  • State-space tokenization avoids compounding errors of action-space approaches (p. 7)
  • Partial-AR conversion is FlashAttention-compatible, practical for deployment (p. 9)

Weaknesses

  • "Unified framework" claim is overstated; four tasks require different engines/interfaces (p. 3)
  • Planning results depend on rule-based policy combinations; GUMP's isolated contribution unclear (Table 4)
  • No variance/confidence reporting; no ablation of KV tokenizer vs. action-space tokenization

Fundamental Limitations and Improvement Directions

Structural Limitations

  • Raster map encoding loses precise geometric information; limits accuracy for map-dependent behaviors
  • State-space vocabulary scales combinatorially as precision or state dimensions increase
  • BEV-only output cannot replace sensor-level simulators for perception-dependent pipelines

Improvement Directions

  • Replace raster encoder with vectorized map encoder (e.g., MapTR) for precise lane geometry
  • Explore continuous diffusion-based decoder to avoid quantization artifacts
  • Multi-dataset joint training (WOD + nuPlan + Argoverse) for better generalization
