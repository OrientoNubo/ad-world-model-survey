Cam4DOcc

Benchmark for Camera-Only 4D Occupancy Forecasting in Autonomous Driving Applications

Ma, Chen, Huang, Xu, Luo, Xu, Gu, Ai, Wang | arXiv 2311.17663 | 2023-11

4D Occupancy Forecasting Camera-Only Autonomous Driving Benchmark

Problem Statement

Existing camera-based occupancy estimation methods predict only the current frame's 3D occupancy. They do not forecast future occupancy states of surrounding objects, which is critical for proactive planning in autonomous driving.

  • LiDAR-based forecasting methods exist but are expensive and sensor-dependent
  • No standardized benchmark for camera-only 4D (3D space + time) occupancy forecasting
  • BEV-based prediction loses vertical information; full 3D occupancy forecasting is needed

Core Contributions

  1. First camera-only 4D occupancy forecasting benchmark (Cam4DOcc) with standardized evaluation protocol covering four distinct tasks
  2. New dataset format reorganizing nuScenes and Lyft-Level5 into nuScenes-Occupancy and Lyft-Level5 with inflated GMO, fine-grained GMO, GSO, and free-space annotations
  3. Four baseline types adapted for camera-only 4D forecasting: static-world, point cloud voxelization, 2D-3D instance-based, end-to-end
  4. OCFNet: a novel end-to-end network that simultaneously predicts present and future occupancy with 3D backward centripetal flow

OCFNet Architecture Overview

6 Surround-View Images (900x1600) | v ResNet50 + FPN (2D Encoder) | v LSS 2D-3D Lifting Module | v 3D-ResNet18 (Voxel Encoder) | v 6-DOF Ego-Motion Compensation + Spatiotemporal Convolutions | v Motion-Aware Feature (F_agg) | v Future State Prediction Module | | v v Occ Head Flow Head (O_t) (3D Backward Centripetal Flow)

Parameters: 370M | GFLOPs: 6,434 | GPU Memory: 57 GB (8x A100)

Key Module: Multi-Frame Feature Aggregation

  • ResNet50+FPN extracts 2D features from 6 surround-view cameras
  • LSS-based lifting transforms 2D features into unified 3D voxel space
  • 3D-ResNet18 encodes voxel features

Temporal alignment uses 6-DOF ego-pose to transform all frames into the current coordinate system. Time and feature dimensions are collapsed for 3D spatiotemporal convolutions, and relative ego-poses between adjacent frames are concatenated to produce motion-aware features.

Key Module: Future State Prediction

  • Voxel encoder produces multi-scale features at 3 resolution levels
  • Stacked residual 3D convolutional blocks expand channels from $C$ to $(N_f + 1) \times C$
  • Voxel decoder with skip connections reconstructs full-resolution occupancy

3D Backward Centripetal Flow (adapted from PowerBEV [22]): each occupied voxel's flow vector points toward the instance centroid, enabling instance-level tracking across time via NMS at $t=0$ and flow-based ID association for $t \in [1, N_f]$.

Loss Function

$$\mathcal{L} = \sum_{t=0}^{N_f} \lambda_1 \mathcal{L}_{occ}(O_t, \hat{O}_t) + \lambda_2 \mathcal{L}_{flow}(D_t, \hat{D}_t) + \lambda_3 \mathcal{L}_{depth}(\mathbf{D}_d, \hat{\mathbf{D}}_d)$$
TermWeightPurpose
$\mathcal{L}_{occ}$$\lambda_1 = 0.5$Cross-entropy for occupancy prediction
$\mathcal{L}_{flow}$$\lambda_2 = 0.05$3D backward centripetal flow prediction
$\mathcal{L}_{depth}$$\lambda_3 = 0.5$Depth reconstruction from 2D-3D lifting

Key Insights and Limitations

Strengths

  • Fills a genuine gap: first standardized benchmark for camera-only 4D occupancy forecasting
  • Comprehensive baseline coverage with four distinct approach types
  • 3D backward centripetal flow enables instance-level temporal association in voxel space

Limitations

  • OCFNet assembles existing modules (LSS, 3D ResNet, PowerBEV flow) without fundamentally new architecture
  • No computational cost or inference latency analysis for any method
  • GMO conflation: all movable objects treated as one category, ignoring class-specific motion patterns

Structural Limitations of the Method

  • Depth estimation bottleneck: LSS-based monocular depth is inherently noisy at long range; depth loss uses LiDAR ground truth during training, creating a train-test distribution gap
  • Linear scaling with forecast horizon: Channel expansion to $(N_f+1) \times C$ makes longer-range predictions prohibitively expensive (370M params, 57 GB for 4 future steps)
  • Fixed temporal resolution: No mechanism for variable-speed dynamics or event-driven prediction

Improvement Directions

  1. Per-class motion modeling: Separate prediction heads for vehicles, pedestrians, cyclists (feasible with existing annotations)
  2. Transformer-based temporal modeling: Replace 3D conv temporal expansion with temporal transformers for non-local dependencies and lower memory cost
  3. Uncertainty-aware prediction: Per-voxel confidence for downstream planning, especially at longer horizons
  4. Sparse voxel structures: Octree or sparse representations to reduce the 57 GB memory footprint
  5. Self-supervised depth: Remove LiDAR dependency in training via temporal self-supervision
