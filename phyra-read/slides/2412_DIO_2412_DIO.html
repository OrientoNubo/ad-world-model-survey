Problem: The Gap Between Instance Interpretability and Geometric Fidelity

  • Instance-based methods: Detect-then-forecast pipelines produce discrete trajectories from bounding boxes, but boxes are coarse approximations of true object geometry; complex shapes (person with arms extended, excavator) cannot be represented (p.1)
  • BEV occupancy methods: Collapse 3D to 2D, losing vertical structure; cannot identify which occupancy belongs to which agent (p.1)
  • Unsupervised 4D models (UnO, 4D-Occ): Predict scene-level occupancy with LiDAR self-supervision, but cannot decompose instances; single-resolution BEV encoding limits fine-grained structure prediction (p.1-2)

Core Idea: Source-Point Prompting for Decomposable Occupancy

DIO frames 4D occupancy as a prompted neural field: given a source point s identifying the object of interest (its 3D location at $t_0$) and a query point q (4D space-time location), DIO predicts the probability that the instance at s will occupy q at time $q_t$, plus a 3D flow vector.

Key design: empty source point s = ∅ produces scene occupancy (all foreground + background); non-empty s produces instance occupancy for that specific object. The same model, the same forward pass, two modes of output (p.3, Sec. 3.1).

Source points can come from any off-the-shelf 3D detector, from GT box centroids, or from a motion planner — the model is trained to be robust to noisy prompts (p.3, 7).

Architecture Overview

Multi-sweep LiDAR (16 frames @ 0.2s, 3.0s history) | v [SECOND Sparse 3D Backbone] Sparse feature volumes: V2x, V4x, V8x, V16x | +---> V2x, V4x, V8x (high-res sparse, limited receptive field) | +---> V16x --> [Densify + Deformable BEV Attention Neck] Dense BEV map: M16x (large receptive field) Scene features Z = [V2x, V4x, V8x, M16x] For each prompt (query point q, source point s): | v [MSI: Multi-Scale Interpolation at q and at s] + sinusoidal positional embeddings | v [MLP] --> 4H learned 3D offsets Δq | v [Multi-Head Deformable Attention] Interpolate Z at {q + Δq_h} per head h at 4 resolutions Cross-attention aggregation | v [MLP] --> occupancy probability o, flow vector f

Training: Prompt Taxonomy (Figure 3, p.5-6)

The training procedure constructs five types of prompts to achieve scene occupancy, instance decomposability, and free-space robustness:

P+scene / P-scenes = ∅Ray-based: positive = 10 cm past LiDAR return; negative = along ray before returnOccupied / Unoccupied

P-box (hard negative)Noisy centroid of biQueries in scaled-up box bi' \ biUnoccupied (encourages instance separation)

Per sample: 19,000 positive + 19,000 negative scene queries; 38,000 instance prompts; 6,000 Pbox + 6,000 Prand (p.5-6).

Loss Function

Binary cross-entropy occupancy loss summed over all prompts, plus a flow consistency loss (Eq. 4-5, p.5):

$$\mathcal{L} = \mathcal{L}_{occ} + \lambda \mathcal{L}_{flow}$$

Flow consistency: given predicted flow $\mathbf{f}$ at query $\mathbf{q}$, compute the displaced point $\mathbf{q}' = (x+f_x\Delta t, y+f_y\Delta t, z+f_z\Delta t, t+\Delta t)$ and enforce that occupancy is consistent across the displacement:

$$\mathcal{L}_{flow} = -\sum_{(\mathbf{q},s) \in P_{all}} \bigl[ o(\mathbf{q},s)\log o(\mathbf{q}',s) + (1-o(\mathbf{q},s))\log(1-o(\mathbf{q}',s)) \bigr]$$

Training: AdamW, lr 1e-4, 1,000-iter warmup from 1e-5, cosine decay, 50,000 iterations, batch size 16 (p.5).

Limitations and Open Questions

  • Instance decomposition requires external proposals: If no source point is available for an object (detector miss), that object is invisible in instance mode; DIO has no autonomous instance discovery mechanism (Conclusion, p.9)
  • Single-dataset evaluation: All main results are on AV2; the open-set experiment covers only three held-out classes; no evaluation on nuScenes or Waymo limits generalizability claims
  • Inference cost unknown: Per-query implicit inference has a fundamentally different throughput profile from voxel-grid methods; dense semantic occupancy at inference (querying full grid with detector source points) has no reported FPS
  • No flow loss ablation: The flow consistency loss is a design choice but its individual contribution to forecasting accuracy is not isolated in any table

Key Takeaways

  • Source-point prompting unifies scene and instance occupancy prediction in one model: empty source point = scene occupancy, non-empty source point = instance occupancy; the same trained weights serve both modes
  • DIO-D (prompted by off-the-shelf HEDNet detector) outperforms Box Oracle on instance mAPI (95.19 vs. 50.49), showing that implicit geometry from LiDAR self-supervision captures true shapes better than ground-truth bounding boxes
  • DIO-NL object-based LiDAR completion L1 is 4.3x smaller than the best prior unsupervised baseline (UnO), with the largest gap in the object-centric metrics where the decomposed representation provides the most value
  • Open-set behavior diverges predictably: DIO-∅ generalizes best at shape completion for novel classes (no semantic encoding); DIO-NL generalizes best at forecasting (source point encodes implicit motion prior)
