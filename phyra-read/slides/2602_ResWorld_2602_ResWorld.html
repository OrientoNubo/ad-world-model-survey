Problem: Why Existing World Models Fall Short

  • Redundant static modeling: Existing world models predict full future scenes, wasting capacity on static objects (roads, buildings) that do not change across timestamps (p.1-2)
  • Weak trajectory-scene interaction: Prior methods lack deep interaction between predicted trajectories and future scene representations, limiting planning accuracy (p.2)
  • World model collapse: Without supervision from auxiliary tasks, latent world models tend to collapse, mapping diverse driving scenes to identical representations (p.5)

Core Idea: Temporal Residuals for Dynamic Objects

Instead of predicting full future scenes, ResWorld computes temporal residuals between consecutive BEV features to isolate dynamic object information. The world model then only needs to predict changes (dynamic objects), while static scene information is directly inherited from the current BEV features.

Key insight: by shifting BEV features from different timestamps into the current coordinate system and subtracting them, the residuals capture only the regions where things moved, naturally filtering out static elements (p.2, p.4).

Architecture Overview

Multi-view Images (t, t-1, ..., t-k) | v [GeoBEV Encoder] --> BEV Features {B_t, B_{t-1}, ..., B_{t-k}} | | v v [TokenLearner] [Temporal Residual Extraction] Fused Scene Query R_t, R_{t-1}, ..., R_{t-k+1} | | v v [Cross Attention] [TR-World: Self-Attention + TokenFuser] Prior Trajectory T_prior Future BEV Feature B_future | | +----------+-------------+ | v [FGTR: Deformable Attention] (W queries attend to B_future guided by T_prior reference points) | v Final Trajectory T_final

Module 1: Prior Trajectory Prediction

Multi-view images are converted to BEV features using GeoBEV (Zhang et al., 2025), then fused via concatenation and convolution (Eq. 1, p.3):

$$\mathbf{B}_{fuse} = \text{Conv}(\text{Concat}(\mathbf{B}_t, \mathbf{B}_{t-1}, \ldots, \mathbf{B}_{t-k}))$$

TokenLearner extracts sparse scene queries $\mathbf{S}_{fuse} \in \mathbb{R}^{N_s \times C}$ via spatial attention pooling (Eq. 2). Waypoint queries $\mathbf{W} \in \mathbb{R}^{N_t \times C}$ decode prior trajectories through cross attention (Eq. 4, p.4):

$$\mathbf{T}_{prior} = \text{MLP}(\text{CrossAttention}(\mathbf{W}, \mathbf{S}_{fuse}, \mathbf{S}_{fuse}))$$

Module 2: Temporal Residual Extraction

For each timestamp $i$, sparse scene queries are extracted using the fused BEV's spatial attention map (Eq. 5, p.4):

$$\mathbf{S}_i = \text{AvgPool}(\text{SA}(\mathbf{B}_{fuse}) \odot \mathbf{B}_i)$$

Temporal residuals are computed by subtracting consecutive scene queries: $\mathbf{R}_i = \mathbf{S}_i - \mathbf{S}_{i-1}$. These residuals represent only the changes caused by dynamic objects, since static objects remain constant across aligned timestamps.

Module 3: Temporal Residual World Model (TR-World)

TR-World processes only temporal residuals through self-attention layers, then accumulates them to form a future dynamic object representation (Eq. 6, p.5):

$$\hat{\mathbf{R}} = \sum_{i=t-k+1}^{t} \text{SelfAttention}(\mathbf{R}_i)$$

TokenFuser maps $\hat{\mathbf{R}}$ back to BEV space and adds it to the fused BEV feature to produce the future BEV (Eq. 7):

$$\mathbf{B}_{future} = \text{TokenFuser}(\hat{\mathbf{R}}, \mathbf{B}_{fuse}) + \mathbf{B}_{fuse} = \text{MLP}(\mathbf{B}_{fuse}) \otimes \hat{\mathbf{R}} + \mathbf{B}_{fuse}$$

This design ensures static information is preserved from $\mathbf{B}_{fuse}$ while only dynamic changes are predicted by the world model.

Module 4: Future-Guided Trajectory Refinement (FGTR)

FGTR applies Deformable Attention between waypoint queries $\mathbf{W}$ and future BEV features $\mathbf{B}_{future}$, using prior trajectory $\mathbf{T}_{prior}$ as reference points (Eq. 8-9, p.5):

$$\mathbf{W} = \text{DeformAttn}(\mathbf{W}, \mathbf{B}_{future}, \mathbf{T}_{prior})$$
$$\mathbf{T}_{final} = \text{MLP}(\mathbf{W})$$

Two functions of FGTR:

  • Checks if prior trajectories collide with objects or leave drivable areas by querying future BEV features, correcting the trajectory
  • Provides sparse spatial-temporal supervision on future BEV features (at reference points), preventing world model collapse without requiring dense future supervision

Loss Function

Only L1 loss on trajectories, no explicit world model supervision (Eq. 10, p.6):

$$\mathcal{L} = \text{L1}(\mathbf{T}_{prior}, \mathbf{T}_{GT}) + \text{L1}(\mathbf{T}_{final}, \mathbf{T}_{GT})$$

The paper explicitly demonstrates that not supervising $\mathbf{B}_{future}$ with real future data yields better performance than direct future supervision (Table 4, p.8). This is because direct supervision at $t+1$ loses temporal information for other timestamps, while the implicit supervision through FGTR preserves cross-temporal spatial information.

Supervising with real future data at timestamp $t+1$ forces the model to lose richer temporal information across multiple future timestamps. The sparse implicit supervision from FGTR's trajectory reference points is more effective, allowing the model to independently optimize and retain the most task-relevant information (p.9).

Limitations and Open Questions

  • Potential dynamic objects: Temporal residuals are zero for currently stationary objects (parked cars, waiting pedestrians). These cannot be captured by the residual mechanism and must be handled by the prior trajectory branch as static objects (p.10)
  • No world model quality evaluation: The paper evaluates planning accuracy but never directly measures the quality of predicted future BEV features (e.g., future occupancy IoU)
  • Camera-only: The framework is designed for camera input; how temporal residuals work with LiDAR or radar BEV features is unexplored

Key Takeaways

  • Temporal residuals in BEV space naturally separate dynamic and static objects without any detection labels, enabling the world model to focus computational effort on what matters for planning
  • Trajectory-guided sparse supervision (via FGTR) is more effective than dense future BEV supervision for preventing world model collapse, because it preserves multi-timestamp temporal information
  • The framework achieves SOTA on both nuScenes and NAVSIM without requiring auxiliary task annotations (detection, tracking, mapping), substantially reducing annotation dependency
