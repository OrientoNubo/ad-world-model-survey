Problem Statement

AV simulation requires both scene initialization (generating diverse initial traffic layouts) and scene rollout (closed-loop agent behavior). Prior approaches treat these as separate problems, and closed-loop diffusion suffers from:

  • Compounding errors when replanning at high frequency (distributional drift)
  • Prohibitive computational cost: full denoising (16 steps) at every physical timestep
  • Limited controllability for generating safety-critical long-tail scenarios

Core Idea: Amortized Diffusion

Consecutive simulation steps share most trajectory content. Instead of regenerating predictions from scratch at every timestep, carry forward partially-denoised predictions and refine with one denoising step per physical timestep.

Full AR (baseline)

  • At each physical step: run complete 16-step diffusion
  • Cost: $T_{\text{phys}} \times 16$ denoising calls
  • Quality degrades at high replan rates (Fig. 5)

Amortized AR (proposed)

  • Warm-up: one full 16-step prediction
  • Each subsequent step: one denoising evaluation
  • Cost: $16 + T_{\text{phys}}$ calls total
  • Better trajectory consistency (Fig. 5, 7)

Architecture

Two end-to-end trained components:

ComponentRoleDetails
Global Context Encoder Compress map + signals to fixed-size tokens Perceiver IO with 256 latent queries (p.4, Fig. 3)
Transformer Denoiser Denoise scene tensor conditioned on context AdaLN conditioning, self-attn (time) + spatial-attn (agents) (p.4, Fig. 3)

Scene tensor: $x \in \mathbb{R}^{A \times T \times D}$ where A = agents, T = timesteps, D = features (position, heading, size, type). All tasks formulated as inpainting with different masks (Fig. 2, p.3).

Unified Task Formulation

A single model handles multiple tasks through different inpainting masks:

TaskWhat is masked (generated)What is given (conditioned)
Behavior PredictionFuture of all agentsHistory of all agents

Conditional SceneGenControlled by per-feature, per-timestep masksSpecified features at specified times

Generalized Hard Constraints (GHC)

Iterative clipping at each denoising step, not just post-hoc:

$$\mu_{t \to s} = \frac{\alpha_{t \to s}}{\sigma_t^2} z + \frac{\alpha_s \sigma_{t \to s}^2}{\sigma_t^2} \text{clip}(x)$$

Three constraint types (p.7, A.9):

  • Non-collision: minimize overlap potential between bounding boxes
  • Range: clamp features to $[d_{\min}, d_{\max}]$ (e.g., vehicle length 7-9m)
  • On-road: minimize offroad potential relative to road graph polylines

In-diffusion GHC produces more realistic layouts than post-diffusion application (Fig. 8, p.7).

Scaling Analysis

Model Size Scaling

Three sizes: S (128d, 2L), M (256d, 4L), L (512d, 8L). Larger models consistently improve scene generation realism (Fig. 6, p.6).

Temporal Resolution Scaling

Patch sizes {8, 4, 2, 1}. Smaller patches = higher temporal resolution. Both axes improve quality; compute scales with GFLOPs.

Evaluation and Limitations

Strengths

  • 16x cost reduction is well-validated
  • WOSAC benchmark provides standardized evaluation
  • Scaling trend is clear and reproducible

Limitations

  • Does not surpass non-diffusion SOTA overall (tied with MVTe at 0.700)
  • Controllable generation: only 40/100 success rate, qualitative evaluation only (p.22)
  • Validity masks not learned; uses log masks + interpolation
  • No wall-clock timing benchmarks

Key Takeaways

  • Amortized diffusion is a principled way to reduce inference cost in autoregressive diffusion rollouts, applicable beyond driving simulation
  • The unified scene tensor + inpainting formulation elegantly handles initialization and rollout with one model
  • GHC (in-diffusion clipping) is more effective than post-hoc constraint enforcement
  • Model and temporal resolution scaling both improve realism, suggesting the approach has room to grow with compute
  • Main gap: controllability evaluation is underdeveloped; the 40% success rate for LLM-generated constraints needs further work
