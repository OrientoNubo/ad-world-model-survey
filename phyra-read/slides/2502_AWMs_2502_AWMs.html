2. Research Overview

AWMs proposes using differentiable simulation (Waymax) to train three distinct state predictors for autonomous driving: a relative odometry predictor, an optimal state planner, and an inverse optimal state estimator. Rather than training a policy, each predictor is embedded in an end-to-end computation graph with the simulator, enabling supervised training without trial-and-error search. The agent is lightweight (6M parameters) and runs in real-time on the WOMD benchmark.

Weaknesses
  1. Framework operates on idealized Waymax vectorized states; no path to raw sensor inputs, limiting comparison to camera/LiDAR-based world models.
  2. Experiments are deliberately separated; no joint comparison against latent or occupancy world models on identical conditions.
  3. MPC improvement from 1 to 8 rollouts is real (~14% overlap reduction) but modest in absolute terms (Table 8).
4. Problem and Motivation
Storyline

Differentiable simulation enabled APG to turn unsupervised policy learning into a supervised problem by back-propagating through vehicle dynamics. AWMs asks: does this benefit extend to world modeling? The authors show that for "off-policy" world modeling tasks (predicting optimal or counterfactual states rather than the log-replay next state), the supervision targets require inverse kinematics or inverse simulator outputs that are analytically unavailable in black-box environments. DiffSim thus converts structurally intractable tasks into efficiently supervised ones.

5.1 APG Policy (Winner-Take-All Fix)

Function: Selects actions that drive the ego-vehicle toward expert states via outcome-space supervision.

$$\min_\theta \| \text{Sim}(s_t, \pi_\theta(s_t)) - \hat{s}_{t+1} \|_2^2$$

Key fix: During training, gradients back-propagate only through the GMM component whose mean is closest to InvKin$(s_t, \hat{s}_{t+1})$ — preventing mode collapse. Result: ADE 1.8121 vs. baseline 2.0083 (Table 2, 9.8% improvement).

5.2 Relative Odometry AWM

Function: Predicts the relative state change from executing a given action, enabling autoregressive imagination of future positions.

$$\min_\phi \| \text{Sim}(s_{t+1} - f_\phi^O(s_t, a_t),\ a_t) - s_{t+1} \|_2^2$$

Minimized when $f_\phi^O(s_t, a_t) = s_{t+1} - s_t$. State space: $(x, y, v_x, v_y, \alpha)$. At 1s horizon: ADE 0.35m with DiffSim vs. 0.79m without (Table 5).

5.3 Optimal Planner AWM

Function: Prescriptive predictor — outputs the desired next state to visit; inverse kinematics converts this to an executable action.

$$\min_\phi \| \text{Sim}(s_t,\ \text{InvKin}(s_t,\ s_t + f_\phi^P(s_t))) - \hat{s}_{t+1} \|_2^2$$

Gradient flows through Sim and InvKin back to $f_\phi^P$. InvKin is unavailable in black-box environments — this task is structurally unsolvable without DiffSim. Result: ADE 1.8734, overlap 0.0719 vs. APG baseline 2.0083 / 0.0800 (Table 6).

5.4 Inverse Optimal State Estimation AWM

Function: Counterfactual predictor — finds the state $\tilde{s}_t$ from which action $a_t$ would have reached the expert next state $\hat{s}_{t+1}$.

$$\min_\phi \| \text{Sim}(s_t + f_\phi^I(s_t, a_t),\ a_t) - \hat{s}_{t+1} \|_2^2$$

Supervision target $\tilde{s}_t = \text{Sim}^{-1}(\hat{s}_{t+1}, a_t)$ is analytically unavailable without DiffSim. The norm $\|f_\phi^I(s_t, a_t)\|_2$ serves as a confidence metric: near zero when the agent is on expert trajectory, large when deviating (Fig. 5).

Fundamental Limitations
  1. Requires white-box differentiable dynamics (Waymax bicycle model); no path to raw sensor inputs — not fixable within the current framework.
  2. Single-agent ego only; other traffic participants replay logged trajectories, eliminating reactive interaction and limiting real-world transfer.
  3. Odometry accumulates error autoregressively: ADE grows from 0.17m at 0.5s to 0.55m at 1.5s (Table 5), limiting effective planning horizon.
Citations Worth Tracking
  1. APG (Nachkov, Paudel, Van Gool 2025, IROS): Direct precursor; prerequisite for understanding all three AWM formulations.
  2. Waymax (Gulino et al. 2024, NeurIPS): The differentiable simulator that makes all AWM training formulations possible.
  3. Wayformer (Nayakanti et al. 2023, ICRA): Source of the winner-take-all GMM idea; key to understanding the Gaussian collapse fix.
  4. MuZero (Schrittwieser et al. 2020, Nature): Canonical model-based action selection context for the MPC experiment.
  5. TrafficBots v1.5 (Zhang, Sakaridis, Van Gool 2024): Same group; Waymo Sim Agents evaluation for positioning AWMs in the Waymo ecosystem.
8. Open Questions and Improvement Ideas
Outstanding Questions
  • Can the odometry predictor be stabilized at horizons beyond 1.5s without architectural changes (e.g., uncertainty-aware prediction)?
  • How does the planner behave during early training when large actions are generated and clipping is disabled?
  • What is the gap when the "without DiffSim" baseline uses a more competitive learned dynamics model (e.g., transformer next-state predictor)?
  • Can AWMs extend to reactive multi-agent settings where other agents are also controlled?
Improvement Directions
  1. Uncertainty quantification for odometry (feasibility: moderate): Add predictive variance (ensemble or Bayesian head) to allow MPC to down-weight high-uncertainty imagined trajectories. Architecturally natural given existing GMM policy.
  2. Reactive multi-agent AWMs (feasibility: low-moderate): Replace log-replay for other agents with a reactive policy; requires gradient decision across agent boundaries.
  3. Sensor-based input bridge (feasibility: low): Connect a differentiable BEV encoder to the AWM framework; significant engineering but would substantially expand real-world applicability.
