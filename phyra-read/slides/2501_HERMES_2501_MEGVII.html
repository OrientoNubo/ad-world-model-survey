HERMES: A Unified Self-Driving World Model for Simultaneous 3D Scene Understanding and Generation

Bridging BEV-Based Scene Generation and LLM-Driven Scene Understanding in a Single Framework

World Model Autonomous Driving BEV LLM Scene Understanding Point Cloud Generation

Xin Zhou*, Dingkang Liang*†, Sifan Tu, Xiwu Chen, Yikang Ding, Dingyuan Zhang, Feiyang Tan, Hengshuang Zhao, Xiang Bai✉
Huazhong University of Science and Technology, MEGVII Technology, Mach Drive, The University of Hong Kong

Core Idea

HERMES is the first unified Driving World Model that simultaneously performs 3D scene understanding (via language) and future scene generation (via point cloud prediction) within a single framework. The key insight: rather than treating understanding and generation as separate tasks with separate models, HERMES uses a shared BEV representation as the common interface between an LLM and a neural renderer.

  1. BEV Tokenizer converts multi-view images into flattened BEV features fed to the LLM
  2. LLM (InternVL2-2B) processes text instructions and generates both textual understanding and world queries
  3. World Queries transfer world knowledge from the LLM to the Render via causal attention, enabling future BEV generation
  4. Shared Render (BEV-to-Point) produces point cloud predictions for both current and future scenes

Architecture Overview

Multi-view Images I_t | v [BEV Tokenizer] --> Flattened BEV F_t (w x h x c, downsampled to L_bev tokens) | v [LLM (InternVL2-2B)] <-- Text Instructions T, World Queries Q^w | | | Text Tokens | Current-to-Future Link v (Understanding) v (cross-attention layers) Scene Description Future BEV B_{t+1}, ..., B_{t+dt} + VQA Answers | v [Shared Render (BEV-to-Point)] | v Point Cloud P_t, P_{t+1}, ..., P_{t+dt}

The BEV Tokenizer uses OpenCLIP ConvNext-L backbone (p.9). The flattened BEV is projected to LLM dimension via two-layer MLP. World queries are derived from max-pooling the flattened BEV $\mathcal{F}_t$ and processed through 3 cross-attention blocks in the current-to-future link module.

Key Technical Components

1. BEV-based World Tokenizer

Adopts BEVFormer v2 architecture. Multi-view images at time $t$ are encoded by CLIP image encoder and converted to BEV features $\mathcal{F}_t^{bev} \in \mathbb{R}^{w \times h \times c}$ with $w = h = 200$ and $c = 256$ (p.9). A downsampling block reduces tokens by $4 \times$ twice, yielding $L_{bev} = \frac{w}{4} \times \frac{h}{4} = 2{,}500$ tokens, well within standard LLM context windows (vs. ~47,000 for raw views).

2. World Queries and Current-to-Future Link

$n = 4$ groups of world queries $\mathcal{Q}^w$ are generated per future timestep, each containing queries from max-pooling the flattened BEV. These queries are appended to the LLM input and processed through causal attention, allowing them to absorb world knowledge from text descriptions. The current-to-future link module employs 3 cross-attention blocks to generate future BEV features conditioned on the encoded BEV and world queries (p.5).

3. BEV-to-Point Render

Compressed BEV features are reshaped to $\mathbb{R}^{w \times h \times z \times \hat{c}}$ with an added height dimension, then processed via 3D convolutions to produce volumetric features. Differentiable volume rendering with an SDF field computes depth for each ray. The transmittance $T_i = \prod_{j=1}^{i-1}(1 - \alpha_j)$ with learnable sigmoid opacity $\sigma_t(x) = (1 + e^{-t'})^{-1}$ (p.4).

Ablation Studies

Understanding-Generation Interaction (Table 2, p.7)

Training on both tasks jointly outperforms either task alone. The unified model (0.645 at 0s) beats separated unification (0.663 at 0s) in generation, while maintaining comparable understanding metrics. The performance gap is minor (0.002 MTEOR), indicating mutual benefit rather than interference.

World Queries (Fig. 3, p.7)

Without world queries, future BEV is generated by directly integrating ego-motion into $\mathcal{B}_t$. Adding world queries improves 3s Chamfer Distance by ~10% and introduces only a ~1% decrease in CIDEr, attributed to added optimization complexity.

BEV Size (Table 5, p.7)

Spatial resolution of $50 \times 50$ improves CIDEr by 7.3% and 0s generation by 10% compared to $25 \times 25$. The authors chose $50 \times 50$ as a trade-off between quality and LLM processing length.

World Query Source (Table 4, p.7)

Max pooling of $\mathcal{F}_t$ outperforms both attention pooling and average pooling, capturing peak responses from the BEV feature map more effectively.

Additional Results

NuScenes-QA (Table S5, p.10)

HERMES achieves 61.9% accuracy on the NuScenes-QA benchmark (camera modality), outperforming LLaVA by 14.5% and CenterPoint+MCAN by 2.4%, without requiring 3D object detection supervision.

Scaling Potential (Table S2, p.9)

Scaling from 0.8B to 1.8B LLM yields consistent gains in both tasks. Further scaling to 3.8B shows marginal improvement, suggesting a saturation point for the current architecture.

Controllable Generation (Fig. S1, p.10)

HERMES generates future point clouds conditioned on ego-motion instructions ("stop" vs. "turn right"), demonstrating potential as a controllable world simulator.

Red Flags and Limitations

No perception task evaluation. The paper claims unified understanding, yet does not evaluate on standard 3D detection or tracking benchmarks. Understanding is limited to captioning and VQA tasks (p.8, Sec. 6 Limitation).
Generation limited to point clouds only. Future scene generation produces only sparse point clouds, not images or dense occupancy. This limits practical applications compared to video-generating world models.
Small-scale LLM and single dataset evaluation. All experiments use a 1.8B LLM and are conducted only on nuScenes/OmniDrive-nuScenes. The 3.8B scaling experiment (Table S2, p.9) shows diminishing returns, but no larger models or diverse datasets are tested.
Future image generation not explored. The authors acknowledge (p.8) that future image modality is "still under exploration," leaving a gap in the claimed "unified" framework.

Key Takeaways

  1. HERMES establishes that BEV representations can serve as a shared interface between LLM-based scene understanding and neural rendering-based scene generation, achieving cross-task knowledge transfer through world queries and causal attention.
  2. The world query mechanism is the core enabler: by max-pooling BEV features and routing them through the LLM's causal attention, world knowledge enriches future generation without adding inference cost at test time.
  3. Using a 1.8B LLM, HERMES matches or exceeds 7B-parameter specialist models on understanding tasks while simultaneously performing competitive generation, suggesting that task unification provides a regularization benefit.

Fundamental limitation: The architecture couples understanding and generation through a single LLM bottleneck. This means generation quality is bounded by the LLM's capacity to encode spatial-temporal information into a fixed number of world query tokens. The current design does not support image-level future generation, limiting its utility as a full world simulator.
