GEM: A Generalizable Ego-Vision Multimodal World Model

Fine-Grained Ego-Motion, Object Dynamics, and Scene Composition Control

World Model Ego-Vision Video Generation Diffusion Controllable Multimodal

Mariam Hassan, Sebastian Stapf, Ahmad Rahimi, Pedro Rezende, et al.
EPFL, University of Bern, Swiss Data Science Center, University of Zurich, ETH Zurich

Core Idea

GEM is a multimodal ego-vision world model that generates future video frames (RGB + depth) given a reference frame, with three independent control levels:

  1. Ego-motion control via trajectory conditioning through cross-attention LoRA
  2. Object-level control via sparse DINOv2 feature tokens with identity embeddings, projected through ObjectNet
  3. Human pose control via skeleton conditioning through PoseNet

Built on Stable Video Diffusion (SVD), trained on 4000+ hours of multi-domain ego-vision data spanning autonomous driving, human activities, and drone flights.

Architecture Overview

Reference Frame --> VAE Encoder --> Latents -------+ | Future Frames ---> VAE Encoder --> Noisy Latents ---+--> Denoiser UNet ---> VAE Decoder --> RGB | | Future Depth ----> VAE Encoder --> Noisy Latents ---+ LoRA x-attn <--- Ego-Trajectories | | DINOv2 Features (sparse, masked) --> ObjectNet ----+--> Input Blocks | Human Pose Skeletons --> PoseNet ------------------+--> Input Blocks | Modality Proj --> Depth

The denoiser $D_\theta$ processes concatenated RGB and depth latents, conditioned on all three control signals simultaneously. A modality output projection layer separates RGB and depth predictions.

Three Control Mechanisms

1. Ego-Motion (Cross-Attention LoRA)

Ego-trajectories (2D metric sequences from SLAM or GT) are Fourier-embedded and injected via LoRA adapters on all cross-attention layers. Lightweight adaptation without modifying base SVD weights.

2. Object-Level (ObjectNet + DINOv2 + Identity Embeddings)

Sparse DINOv2 feature maps (randomly masking to $m$ of $M=32$ tokens) encode object semantics. Learned identity embeddings $\text{ID}_\phi$ disambiguate moving vs. inserting objects. Optical flow warps tokens to target frames. ObjectNet projects features into denoiser input blocks.

3. Human Pose (PoseNet)

17-keypoint skeletons drawn on empty planes, encoded by CNN (PoseNet), added to UNet input blocks. Spatial information requires input-block injection rather than cross-attention (confirmed by ablation in supplementary Q4).

Autoregressive Long-Horizon Generation

A dynamic per-frame noise schedule enables stable generation up to 150 frames:

  1. Initialization: All frames begin denoising; frame $i$ starts only after frame $i-1$ completes
  2. Autoregressive: Fully denoised frames are saved and replaced by new noisy frames in a sliding window
  3. Termination: No new frames appended; remaining frames are progressively denoised

Training noise schedule: $t_{\text{shift}} \sim \text{Beta}(\alpha, \beta)$ ensures consistent noise increase over the frame axis. GEM maintains lower FVD and FID than Vista across all tested video lengths (25-150 frames), with the gap widening at longer durations (Fig. 5, p.7).

Red Flags and Limitations

Cross-domain generalizability claimed but not quantitatively verified. Human activity and drone flight evaluations are qualitative only (supplementary Figs. 11-14), despite "Generalizable" appearing in the paper title.
COM metric limitations acknowledged by the authors themselves. The proposed COM metric measures only the largest vehicle's bounding box center offset, which the authors admit is "not an ideal metric" for evaluating identity embeddings (p.19).
Limited baseline comparison on OpenDV. Only Vista provides results on OpenDV; other baselines (DriveDreamer-2, GenAD, etc.) are compared only on Nuscenes (Table 2, p.7).

Key Takeaways

  1. GEM demonstrates that three-level controllability (ego-motion, object, pose) is achievable in a single world model by composing LoRA conditioning, sparse feature injection, and skeleton encoding on top of SVD.
  2. The autoregressive noise schedule is effective: GEM maintains temporal consistency up to 150 frames where baseline methods degrade.
  3. The 4000+ hour multi-domain dataset with pseudo-labels is a practical contribution that could benefit other ego-vision research.

Fundamental limitation: All control signals derive from pseudo-labels, establishing an inherent accuracy ceiling. Object control via sparse DINOv2 tokens provides semantic placement but lacks geometric precision for shape or orientation control.
