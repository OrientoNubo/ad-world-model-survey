TrafficBots

Towards World Models for Autonomous Driving Simulation and Motion Prediction

Zhejun Zhang, Alexander Liniger, Dengxin Dai, Fisher Yu, Luc Van Gool
ETH Zurich / Toyota Motor Europe / KU Leuven | ICRA 2023 | arXiv 2303.04116

Autonomous Driving Traffic Simulation World Model Motion Prediction Multi-Agent Policy CVAE

Core Idea

TrafficBots formulates data-driven traffic simulation as a world model problem: a multi-agent policy that predicts the next states of all non-ego agents given the current scene. Each bot agent is conditioned on two private attributes:

  • Destination (navigational goal): "where to go," predicted as a categorical distribution over map polylines
  • Personality (driving style): "how to drive," modeled as a time-invariant latent variable via CVAE

This design enables a single model to serve both closed-loop simulation and open-loop motion prediction, with configurable and scalable agent behavior.

Architecture Overview

[HD Map + Traffic Lights + Agent States] | v [Transformer State Encoder] (polyline subgraph + map + traffic light + interaction attention) | +---------+---------+ | | v v [Destination [Personality Encoder Predictor] (CVAE: z_post / z_prior)] | | +-------+-----------+ | v [Combine: context + destination + personality] | v [Action Head (2-layer MLP)] -> acceleration, yaw rate | v [Differentiable Dynamics Model] -> next states s_{t+1} | v [Autoregressive rollout for T_f steps]

The entire pipeline is differentiable and fully data-driven. The dynamics model uses kinematic bicycle model for vehicles, single-track for cyclists, and point-mass for pedestrians (Appendix B, p.8).

Key Technical Components

1. State Encoder (Transformer)

Processes shared vectorized context via four attention layers: polyline subgraph (within polyline), map (cross-attention to HD map), traffic lights, and interaction (agent-to-agent). Uses a novel positional encoding that concatenates position, direction, and radian angle (Eq. 3-4, p.3-4).

2. Personality (CVAE)

The posterior encoder sees the full episode (including future) and produces parameters of a Gaussian; the prior encoder sees only past observations. KL divergence with free nats (0.01) prevents posterior collapse. Personality is sampled once per episode and remains fixed, capturing time-invariant driving style (p.4, p.10).

3. Destination Predictor

Multi-class classification over map polylines. GT destinations are approximated via heuristics (extending last observed position along lanes). Cross-entropy loss trains the predictor. At inference, destinations are sampled from the categorical distribution (p.4, Appendix C).

Training Objective

Three loss components:

  1. Reconstruction loss: weighted sum of smoothed L1 on (x,y) positions, cosine distance on yaw angle $\theta$, smoothed L1 on velocity (p.10)
  2. KL divergence: $KL(z_{\text{post}} \| z_{\text{prior}})$ with free nats = 0.01, training the prior to match the posterior while preventing collapse (p.10)
  3. Cross-entropy for destination: maximizes log-likelihood of the GT destination polyline index (p.10)

Training uses counterfactual simulation (rollout with GT future of other agents) and a posteriori simulation (full rollout). Models are trained for 24K iterations (48 hours) on 6x 2080Ti GPUs (p.10).

Strengths

  1. Unifies traffic simulation and motion prediction under one multi-agent policy framework
  2. Destination + personality conditioning provides interpretable, configurable control levers
  3. Fully differentiable pipeline enables potential end-to-end planner training
  4. Open-sourced with competitive WOMD leaderboard performance

Weaknesses

  1. Evaluated on WOMD only; cross-dataset generalization is not demonstrated
  2. Personality latent is not analyzed for interpretability; users cannot meaningfully manipulate it
  3. GT destination approximation via heuristics introduces training noise for complex maneuvers
  4. Motion prediction performance (minADE/minFDE) trails specialized open-loop methods

Contribution Assessment

Claimed ContributionAssessment
Unified multi-agent policy for simulation and prediction Supported: same model evaluated on both tasks with systematic ablation. Trade-off: prediction accuracy lower than specialized methods.
Configurable behavior via destination + personality Partially supported: ablation confirms both components help, but personality interpretability is not demonstrated.
Differentiable data-driven world model Architecturally supported: pipeline is differentiable. But downstream gradient use (e.g., planner training) is not demonstrated, so the "world model" claim remains proof-of-concept.

Fundamental Limitations

  • Autoregressive error accumulation: rollout design inherently accumulates errors over time; this is structural and cannot be fixed by scaling
  • GT destination approximation: heuristic construction from observed trajectories introduces systematic noise for lane-changing or complex maneuvers
  • Single-mode personality per rollout: cannot represent within-episode behavioral shifts (e.g., driver becoming aggressive after being cut off)
  • No explicit multi-agent coordination: each agent acts independently given shared context; cooperative behaviors (e.g., merging) are not modeled

Key Takeaways

  1. Casting traffic simulation as a world model is a productive framing that naturally unifies simulation and prediction tasks
  2. Structured conditioning (destination + personality) is more effective than unstructured latent variables for controlling agent diversity; the ablation provides clear evidence for this
  3. The trade-off between autoregressive realism (low overlap, low traffic rule violations) and displacement accuracy (minADE/minFDE) is an open challenge for the field
