WorldVLM: Combining World Model Forecasting and Vision-Language Reasoning

Stefan Englmeier & Katharina Winter (equal contribution) · Fabian B. Flohr — Munich University of Applied Sciences, IVL

Short name: WorldVLM   |   YYMM: 2603

Autonomous Driving World Model VLM Trajectory Prediction Behavior Conditioning nuScenes
Abstract

Autonomous driving systems depend on models that can reason about high-level scene contexts and accurately predict the dynamics of their surrounding environment. VLMs offer strong capabilities in contextual reasoning but limited spatial comprehension. World Models internalize environmental dynamics to predict future scene evolution. To leverage the complementary strengths of context-based decision making and prediction, the authors propose WorldVLM: a hybrid architecture that unifies VLMs and WMs. The high-level VLM generates behavior commands to guide the driving WM, enabling interpretable and context-aware actions. The paper evaluates conditioning strategies and provides insights into hybrid design challenges.

Highlights
  • Conditioning ablation (Table IV) systematically compares motion vector, discrete speed/goal, and action token representations; speed-only retains most benefit while angle-only degrades L2 to 1.18 at 3s.
  • nuScenes justification-action annotation dataset (22,516 train / 5,615 val) distilled from DriveLM + doScenes via GPT-OSS-120B; stated to be released.
  • Token representation ablation (Table V): first-16-tokens yields lowest MAE (angle 0.0135, speed 0.1788) vs. dedicated behavior tokens or last-n-tokens.
Weaknesses
  • No closed-loop evaluation; open-loop L2 on nuScenes does not measure real driving safety.
  • VLM inference ~1 s vs. WM 12 Hz — irreconcilable frequency mismatch for real-time use.
  • NLP ground truth is LLM-distilled, not human-annotated; BLEU/ROUGE scores measure alignment with noisy reference.
  • Single front-view image only; multi-view temporal context unused by VLM despite being available in nuScenes.

Problem and Motivation

Storyline

Autonomous driving requires both high-level semantic reasoning and spatially accurate trajectory prediction. VLMs provide the former through internet-scale pre-training but fail at precise spatial trajectory generation due to 2D image pre-training and causal confusion [1][2][3]. World models provide the latter through latent-space scene evolution modeling but lack any semantic reasoning layer [6]. The paper argues these two gaps are complementary and proposes connecting them via a behavior command bottleneck, letting each module do what it does best.

Methodology

Framework Overview
[Front-view image + nav instruction + ego-speed] | v [VLM (LLaVA-Qwen)] |--- free-text: justification + action description |--- behavior head (3-layer MLP) ---> [2D behavior vector: (alpha, v)] | [Multi-view images] -> [Spatial Encoder (LAW)] -> [Spatial View Features] | [Behavior vector] --(concat)-->[Waypoint Queries] | [Waypoint Transformer Decoder] | [Predicted Waypoints (ego-traj)] | [Behavior vector + Spatial Features + Waypoints] | [WM Transformer Decoder] | [Future Visual Latent Features]
Module: Behavior-Planning VLM

Input: single front-view image + navigation enum + ego-speed scalar

Output: JSON {justification, action, action_token} + 2D behavior vector (angle, speed)

Behavior head: 3-layer MLP, ReLU, dropout 0.5, input = first 16 output token hidden states

Training: LR 1e-6 → 1e-7, 4 epochs, batch 3, 2× A40 GPU

Loss: $\ell = \text{MSE}(\hat\alpha,\alpha) + \text{MSE}(\hat v, v) + \ell_{\text{text}}$

Module: Behavior Conditioning in LAW

Injection point 1: behavior vector (zero-padded to 8-dim) concatenated with learned waypoint queries before waypoint transformer decoder

Injection point 2: behavior + spatial features + predicted waypoints concatenated before WM transformer decoder (via action-aware MLP)

Ablation (Table III): dual concatenation reduces 3s collision from 0.49% to 0.10% with GT vectors.

Key Formulas

Behavior target derivation from GT trajectory (first point $p_f$, last point $p_l$ over 3 s horizon):

$$v = \frac{d}{30}, \quad \alpha = \frac{\text{atan2}(\Delta p_y, \Delta p_x) + \pi}{2\pi} - 0.75$$ $$\ell = \underbrace{\text{MSE}(\hat\alpha,\alpha) + \text{MSE}(\hat v, v)}_{\ell_{\text{behavior}}} - \underbrace{\frac{1}{T-1}\sum_{t=1}^{T-1}\log p_\theta(y_{t+1}|y_{\le t},x)}_{\ell_{\text{text}}}$$

Fundamental Limitations
VLM inference bottleneck: ~1 s VLM vs. 12 Hz WM — frequency mismatch makes real-time control infeasible under current design.
Single-frame front-view only: LLaVA crops the image; multi-view and temporal context that LAW's encoder already uses are unavailable to the VLM.
Behavior bottleneck information loss: Compressing scene context to a 2D (angle, speed) vector discards all semantic richness; free-text reasoning is used only for NLP metrics, not for WM conditioning.
LLM-distilled annotation quality: BLEU/ROUGE measure similarity to GPT-OSS-120B outputs, which have not been human-validated.

Open Questions

Unresolved Questions
  • Can VLM-predicted behavior vectors (not GT) improve L2/collision over baseline when both stages are fine-tuned jointly?
  • Does reasoning quality (justification accuracy) correlate with trajectory quality, or are the two objectives orthogonal?
  • What is the minimum VLM update frequency needed for behavior commands to remain useful given nuScenes scene dynamics?
  • Would a higher-dimensional latent embedding (e.g., full last-layer hidden state) improve WM conditioning at the cost of interpretability?
Improvement Directions
  1. Joint end-to-end fine-tuning (high feasibility): current two-stage training optimizes VLM and WM separately; joint training may align behavior prediction with trajectory quality.
  2. Human-annotated reasoning traces (medium feasibility): would make NLP metric evaluation meaningful and improve reasoning calibration.
  3. VLM directly connected to WM encoder latents (medium, architectural): eliminates single-frame crop limitation and gives VLM multi-view spatial context.
  4. Closed-loop evaluation in CARLA (high, infrastructure): the key missing experiment; without it, claimed safety improvements cannot be substantiated.
