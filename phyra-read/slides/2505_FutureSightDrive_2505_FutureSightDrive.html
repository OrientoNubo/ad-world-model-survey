FutureSightDrive

Thinking Visually with Spatio-Temporal CoT for Autonomous Driving

Shuang Zeng, Xinyuan Chang, Mengwei Xie, Xinran Liu, Yifan Bai, Zheng Pan, Mu Xu, Xing Wei, Ning Guo

Xi'an Jiaotong University & Amap / DAMO Academy, Alibaba Group

NeurIPS 2025 arXiv: 2505.17685 End-to-End Autonomous Driving Visual CoT World Model

Problem: The Modality Gap in VLA Reasoning

Vision-Language-Action (VLA) models for autonomous driving rely on textual Chain-of-Thought (CoT) to reason about driving decisions. This forces rich visual information through a text bottleneck, causing three problems:

  1. Text CoT compresses spatial relationships into abstract symbols, losing fine-grained visual details (e.g., "pedestrian on the right" discards exact position)
  2. Image-text CoT combines generated images with text descriptions, but the two modalities are often inconsistent with each other
  3. The modality gap between perception (visual) and planning (text) degrades trajectory quality

Core Idea: Think Visually, Not Textually

FSDrive proposes Spatio-Temporal CoT: instead of converting visual observations into text for reasoning, the VLA generates a unified future frame as an intermediate visual reasoning step, then plans trajectories from this visual context.

The same VLA serves dual roles: a world model that imagines the future, and an inverse dynamics model that plans from that imagined future.

Method: Unified Pre-training Paradigm

FSDrive builds on Qwen2-VL-2B without architectural changes. The only modification is expanding the MLLM's vocabulary with VQ-VAE (MoVQGAN) codebook tokens, enabling autoregressive image generation through next-token prediction.

Understanding Branch

VQA tasks on OmniDrive-nuScenes to preserve semantic understanding:

$A = \mathcal{M}(I, L)$

Generation Branch

Future frame prediction via autoregressive visual token generation:

$P(q_1, \ldots, q_{h \cdot w}) = \prod_{i=1}^{h \cdot w} P_\theta(q_i \mid q_{<i})$

Data efficiency: activates visual generation with only ~0.3% of the data used by prior methods (p.3), requiring no training from scratch.

Method: Progressive Generation Curriculum

Directly generating complete future scenes violates physical laws. FSDrive introduces a progressive, easy-to-hard generation approach:

Lane Dividers $Q_l$
Road skeleton, static constraints
→
3D Bounding Boxes $Q_d$
Dynamic objects, motion patterns
→
Full Future Frame $Q_f$
Conditioned on $Q_l$ and $Q_d$

$P(Q_f \mid Q_l, Q_d) = \prod_{i=1}^{h \cdot w} P_\theta(q_i \mid q_{<i}, Q_l, Q_d)$

This curriculum enforces physical plausibility by generating structural priors before fine-grained details.

Method: Spatio-Temporal CoT for Planning

During fine-tuning, the model generates a unified future image frame as the CoT, combining lane dividers (red lines), 3D detection boxes, and the predicted scene into a single image. Visual prompting techniques (red circles, bounding boxes) direct model attention.

$P(W_t \mid I_t, Q_{CoT}, opt(T_{com}, T_{ego})) = \prod_{j=1}^{m} P_\theta(w_t \mid w_{<t}, I_t, Q_{CoT}, opt(T_{com}, T_{ego}))$

This eliminates semantic gaps from cross-modal conversion. The VLA reasons end-to-end in the visual domain: perception, imagination, and planning all operate on image representations.

Training Strategy

Stage 1: Unified Pre-training

  • VQA tasks (OmniDrive-nuScenes)
  • Future frame prediction (nuScenes unlabeled)
  • Progressive generation with annotated CoT data (lane dividers + 3D detection)
  • All understanding and generation tasks trained together

Stage 2: Supervised Fine-tuning

  • Trajectory planning (nuScenes, following VAD)
  • Scene understanding (DriveLM GVQA)
  • Spatio-temporal CoT as intermediate reasoning
  • Single model, task-specific prompts

Implementation: Qwen2-VL-2B base, 32 epochs pre-training, 12 epochs fine-tuning on 8x NVIDIA RTX A6000, batch size 16, learning rate $1 \times 10^{-4}$ (p.7).

Strengths and Limitations

Strengths

  • Clean formulation: single VLA as both world model and planner
  • 31% collision reduction from visual CoT (Table 6)
  • Data-efficient: ~0.3% of prior methods' data
  • Model-agnostic: applicable to any MLLM
  • SOTA on NAVSIM (85.1 PDMS)

Limitations

  • Front-view only (no surround generation)
  • Low generation resolution (128x192)
  • No inference latency reported
  • Single future frame, no multi-step prediction
  • VQ-VAE bottleneck limits visual fidelity

Phyra's Verdict

FSDrive presents a well-motivated departure from text-based reasoning in autonomous driving VLAs. The spatio-temporal CoT concept is the paper's primary contribution, and the 31% collision reduction validates its effectiveness empirically.

The technical execution assembles existing components (VQ-VAE tokenization, autoregressive generation, VQA pre-training) into a coherent system rather than introducing fundamentally new modules. The progressive generation curriculum is the most novel technical element.

The paper's silence on inference latency is a notable gap: if the visual CoT generation takes hundreds of milliseconds, the method may not be viable for real-time driving. The front-view-only limitation is acknowledged and presents a clear path for future work.
