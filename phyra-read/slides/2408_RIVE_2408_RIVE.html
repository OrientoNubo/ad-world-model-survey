Core Contribution

First platform to combine realistic traffic simulation, diffusion-based photo-realistic image generation, and closed-loop AD agent evaluation in a single modular framework.

Key insight: Decoupling traffic logic (Traffic Manager) from visual generation (World Dreamer) allows each component to be optimized independently while enabling closed-loop interaction through their interface.

  • Traffic Manager: realistic traffic flow on any OpenStreetMap road network (p. 3-4)
  • World Dreamer: conditional diffusion model with multi-view consistency and auto-regressive temporal coherence (p. 4-5)
  • Modular architecture: any vision-based AD agent can be plugged in as a "player" (p. 3)

System Architecture

City HD Map (OSM) ──► Traffic Manager ──► Map & Vehicle Layouts ▲ │ │ ▼ Ego Trajectory World Dreamer (Diffusion) ▲ │ │ ▼ Driving Agent ◄────────── Multi-view Images (e.g. UniAD) Prompt (weather, style) ─────────────────► World Dreamer

The closed loop: images are fed to AD agent, which outputs ego trajectory, fed back to Traffic Manager, which updates the scene for the next World Dreamer generation cycle (Figure 2, p. 3).

Traffic Manager

  • Built on LimSim [23], supports any OpenStreetMap road network
  • Hierarchical multi-vehicle decision-making with cooperation factor and trajectory weighting (p. 3-4)
  • Operates at 10 Hz; sends layout updates to World Dreamer every 0.5s (p. 7)

Two modes: Open-loop (ego controlled by Traffic Manager, trajectory recorded) and Closed-loop (ego controlled by AD agent, background vehicles by Traffic Manager)

Road categories: lane boundary, lane divider, pedestrian crossing, drivable area (p. 7). Supports 4 demonstration maps: Singapore, Boston, and CARLA Town05 (p. 7).

World Dreamer

  • Diffusion-based (Stable Diffusion v1.5 + ControlNet) conditional generation (p. 4)
  • Conditional inputs: BEV map, 3D bounding boxes, text prompt, camera parameters, ego pose, reference image (p. 4)
  • Cross-view attention for multi-camera consistency [48] (p. 4)

Auto-regressive generation: Previously generated images + relative ego pose serve as reference conditions, enabling temporal coherence over infinite-length sequences (p. 5). No explicit temporal consistency metric is reported.

Training: nuScenes 700 scenes, AdamW lr=1e-4, batch 4x8, 80K iterations on NVIDIA 4090 (p. 7). Output upscaled to 900x1600 via super-resolution (p. 7).

Evaluation Metrics

PDM Score (PDMS) - Open-loop

$$\text{PDMS}_t = \left(\prod_{m \in \{\text{NC},\text{DAC}\}} \text{score}_m\right) \times \frac{\sum_{w} \text{weight}_w \times \text{score}_w}{\sum_{w} \text{weight}_w}$$

Sub-scores: No Collisions (NC), Drivable Area Compliance (DAC), Ego Progress (EP), Time-to-Collision (TTC), Comfort (C). Adapted from NAVSIM [44] (p. 6).

Arena Driving Score (ADS) - Closed-loop

$$\text{ADS} = R_c \times \text{PDMS}$$

$R_c \in [0,1]$: route completion percentage. Simulation terminates on collision, off-road, or route finish (p. 6).

Fundamental Limitations

  • Speed bottleneck: World Dreamer denoises at 0.5 Hz, making real-time simulation impossible under current diffusion-based architecture
  • Single training distribution: Trained only on nuScenes; generation quality on non-nuScenes geometries is unmeasured
  • Confounded evaluation: Low ADS (0.086) conflates World Dreamer artifacts, Traffic Manager difficulty, and UniAD planning failures

Improvement Directions

  • Replace diffusion with faster generators (consistency models, latent flow) for higher simulation frequency
  • Test multiple AD agents to disentangle platform quality from agent quality
  • Add quantitative temporal consistency metrics (FVD) for long sequences
  • Integrate adversarial/safety-critical scenario generation into Traffic Manager
