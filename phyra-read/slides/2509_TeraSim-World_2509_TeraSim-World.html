TeraSim-World

Worldwide Safety-Critical Data Synthesis for End-to-End Autonomous Driving

Jiawei Wang, Haowei Sun, Xintao Yan, Shuo Feng, Jun Gao, Henry X. Liu

University of Michigan, Tsinghua University, NVIDIA, University of Toronto

Autonomous Driving Safety-Critical Data Synthesis Video Generation Agent Simulation Geographically Grounded

Problem Statement

End-to-end autonomous driving requires extensive, diverse, safety-critical training data. Current approaches face a trilemma:

  • Real-world collection is costly and geographically limited; safety-critical events are rare in naturalistic data (p.1)
  • Existing simulators (CARLA, Mcity) lack visual realism and geographic diversity, creating a sim-to-real gap (p.5-6)
  • Video generation models produce visually plausible outputs but lack controllable agent behaviors and safety-critical scenarios (p.1)

TeraSim-World bridges agent simulation and sensor simulation into a unified, geographically grounded pipeline that produces photorealistic multi-view driving videos with adversarial scenarios from any world coordinate.

Pipeline Architecture

[World Coordinate (lat, lon)] | +---> Map Retrieval (OSM) ---> Road Network (SUMO / OpenDRIVE / Waymo) | +---> Street View (Google Maps API) ---> VLM Prompt (GPT-4o) | +---> Traffic Demand (TomTom API) ---> Flow Parameters | v [TeraSim Agent Simulation] Normal (rule-based / data-driven) + Adversarial (NADE orchestrator) | v [HDMap Video Rendering] ---> 6-view projected 2D HDMap frames | v [Cosmos-Transfer1/7B] ---> Front-view photorealistic video | v [Cosmos-7B/Single2Multi] ---> 6-camera synchronized multi-view video | v [Output: Safety-critical E2E driving data]

Every module is independently replaceable. The pipeline requires no manual configuration for generating normal or adversarial scenarios (p.1-2).

Core Technical Components

Map Retrieval and Conversion (p.3)

  • Input: latitude/longitude coordinates
  • Automatic retrieval from OpenStreetMap with conversion to SUMO, OpenDRIVE, or Waymo formats
  • Supports three behavior modeling paradigms depending on map format

Traffic Agent Simulation (p.3-4)

  • Extended TeraSim framework with normal + adversarial behavior generation
  • Adversity orchestrator: injects cut-in, fail-to-yield, red-light running, pedestrian conflicts
  • Behavior options: rule-based (SUMO), data-driven (SMART, VBD), or NADE-style adversarial

Video Generation (p.4-5)

  • HDMap rendering: 3D cuboid scene projected to 6 camera views using RDS-HQ parameters (Table I)
  • Front-view: Cosmos-Transfer1/7B-Sample4V, 701x1281, 24 FPS, 120 frames
  • Multi-view: Cosmos-7B/Single2Multi-Sample4V, 576x1024 per view, 6 synchronized streams

Red Flags

No quantitative evaluation (p.5-7): All results are qualitative case studies. No FID, FVD, LPIPS, or downstream task metrics are reported. The paper cannot demonstrate that generated data is useful for training E2E models.
No baseline comparison (entire paper): No comparison with DiffScene, Safe-Sim, NADE, or any competing data synthesis pipeline on shared benchmarks.
"Worldwide" claim at minimal scale (Table II, p.7-8): Only 5 locations are demonstrated. No systematic analysis of failure modes across diverse geographies, road types, or API coverage gaps.
No ablation studies: No experiments showing which pipeline components contribute most to output quality or downstream utility.

Strengths

  • Geographic flexibility: Any world coordinate as input, leveraging real-world APIs for geospatial grounding (p.1-2)
  • Modular architecture: Each component (map, demand, agent, video) can be independently upgraded without changing the pipeline (p.1)
  • Adversarial integration: Safety-critical events generated automatically via NADE orchestrator, covering fail-to-yield, red-light running, pedestrian conflicts (p.4, Fig. 5-6)
  • Multi-view output: 6 synchronized camera streams matching real AV sensor configurations at 24 FPS (p.5)
  • Clear writing: Well-structured pipeline description with effective figures (Fig. 1-6)

Limitations and Open Questions

  • Video quality ceiling: Bounded by Cosmos-Drive capabilities; artifacts propagate to downstream training
  • API dependency: Google Street View and TomTom coverage gaps in rural/developing regions are not addressed
  • Open-loop only: Generated videos cannot respond to E2E model actions; closed-loop evaluation remains future work (p.8)
  • Statistical validity: No validation that adversarial scenarios match real-world crash distributions

Key Follow-up Questions

  • What is the generation success rate across different geographic regions and road types?
  • Does training on TeraSim-World data improve E2E driving model performance?
  • What is the computational cost per scenario from coordinate input to final video?
