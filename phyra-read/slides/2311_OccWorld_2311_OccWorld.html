OccWorld: Learning a 3D Occupancy World Model for Autonomous Driving

Wenzhao Zheng, Weiliang Chen, Yuanhui Huang, Borui Zhang, Yueqi Duan, Jiwen Lu

Tsinghua University | arXiv: 2311.16038 | 2023-11-27

Code: github.com/wzzheng/OccWorld

World Model 3D Occupancy Autonomous Driving Generative Transformer

Problem and Motivation

Autonomous driving requires forecasting how the 3D scene evolves for safe decision-making. Existing methods predict object bounding box movements, which fail to capture fine-grained scene structure (road surfaces, vegetation, static infrastructure).

The conventional perception-prediction-planning pipeline treats stages separately, requiring ground-truth labels at each stage and losing information across module boundaries (p.1-3).

Why 3D Occupancy?

  • Expressiveness: Describes fine-grained 3D structure of the full scene, not just detected objects
  • Efficiency: Can be learned from sparse LiDAR points, more economical than dense point clouds
  • Versatility: Adapts to both vision and LiDAR inputs; a unified representation

Core Idea

Build a world model in 3D semantic occupancy space that jointly predicts the future evolution of the surrounding scene and the ego vehicle trajectory, using a GPT-like autoregressive generation paradigm.

The world model $w$ is formally defined as:

$$w(\{\mathbf{y}^T, \cdots, \mathbf{y}^{T-t}\}, \{\mathbf{p}^T, \cdots, \mathbf{p}^{T-t}\}) = \mathbf{y}^{T+1}, \mathbf{p}^{T+1}$$

Given past occupancy observations $\mathbf{y}$ and ego positions $\mathbf{p}$, predict the next scene and ego displacement in an autoregressive manner (p.3, Eq.3).

Architecture Overview

OccWorld consists of two main components, trained in two stages:

Stage 1: Train Scene Tokenizer [3D Occupancy] -> [CNN Encoder] -> [VQ-VAE Codebook] -> [Discrete Scene Tokens] | [Decoder] <- ---+-> [Reconstructed Occupancy] Stage 2: Train Spatial-Temporal Generative Transformer [Scene Tokens t=0..T-1] -> [Spatial Aggregation (multi-scale)] -> [Temporal Causal Self-Attention] -> [U-Net Aggregation] -> [Predicted Tokens t=T] | [Scene Decoder] <-----+----> [Future 3D Occupancy] [Ego Decoder] <-- [Ego Token] --> [Ego Displacement]

Module 1: 3D Occupancy Scene Tokenizer

Compresses 3D semantic occupancy into compact discrete tokens using a VQ-VAE approach (p.4, Fig.3).

  • Converts semantic occupancy to BEV via learnable category embedding + height concatenation
  • CNN encoder with 2D convolutions produces downsampled features
  • Vector quantization maps each spatial feature to the nearest codebook entry: $x_q = \arg\min_{c_j \in C} \|k_i - c_j\|_2$
  • Decoder reconstructs occupancy from quantized tokens

Training loss (first stage):

$$J_{e,d} = L_{\text{soft}}(d(e(\mathbf{y})), \mathbf{y}) + \lambda_1 L_{\text{lovasz}}(d(e(\mathbf{y})), \mathbf{y})$$

Best configuration: codebook size 512, 128-dim features, spatial resolution $G^2$ (Table 3, p.8).

Module 2: Spatial-Temporal Generative Transformer

A multi-scale GPT-like architecture for predicting future world tokens (p.4-5, Fig.4).

  • Spatial aggregation: Self-attention within each time frame captures spatial dependencies; tokens merged in 2x2 windows for multi-scale hierarchy
  • Temporal causal attention: Masked attention across time steps prevents future information leakage: $z_{j,t}^{i+1} = \text{TA}(z_{j,1}^i, \cdots, z_{j,t}^i)$
  • U-Net structure: Aggregates predictions across K spatial scales for consistency

Training loss (second stage, p.5, Eq.8):

$$J_{w,d_{ego}} = \sum_{t=1}^{T}\sum_{j=1}^{M_0} L_{\text{soft}}(\hat{z}_{j,0}^t, \mathbf{C}(\hat{z}_{j,0}^t)) + \lambda_2 L_{L2}(d_{ego}(\hat{z}_0^t), \mathbf{p}^t)$$

Critical Assessment

Contributions Verified

  • The occupancy-based world model framework genuinely operates on voxel grids and demonstrates scene-level forecasting (Tables 1-2)
  • Versatility across input modalities (GT, dense pred, LiDAR, camera) is demonstrated via four variants

Claims That Exceed Evidence

  • "Competitive planning without instance/map supervision" is overclaimed: OccWorld-O still uses map and box supervision; OccWorld-S (truly unsupervised) achieves L2 1.83m, not competitive with UniAD's 1.03m
  • "Efficiency" of 3D occupancy: OccWorld-D/T run at only 2.8 FPS, slower than VAD-Tiny at 16.8 FPS
  • "More reasonable drivable areas than GT" is qualitative only; no metric validates this (Fig.1)

Structural Limitations

  • Two-stage training creates an information bottleneck; tokenizer errors cannot be corrected by downstream gradients
  • No explicit object interaction modeling limits collision avoidance capability
  • Evaluated only on nuScenes; generalizability is unverified
