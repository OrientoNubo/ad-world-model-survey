MaskGWM

A Generalizable Driving World Model with Video Mask Reconstruction

Jingcheng Ni, Yuxin Guo, Yichen Liu, Rui Chen, Lewei Lu, Zehuan Wu

SenseTime Research | arXiv: 2502.11663 | 2025-02-17

Core Idea

Diffusion generation and mask reconstruction are complementary tasks. Diffusion handles fine-grained generation quality; mask reconstruction strengthens contextual understanding and generalization. Combining them improves both fidelity and temporal consistency.

MaskGWM = DiT backbone (SD3, 2B params) + Rectified Flow + Video Mask Reconstruction as complementary training task

Three key designs: (1) Diffusion-related mask tokens bridging MR and generation, (2) Separate spatial/temporal masking with two-branch reconstruction, (3) Row-wise shifted temporal self-attention for efficiency.

Method Overview

Architecture: DiT encoder $E$ + Spatial/Temporal/View transformer blocks + Token Reconstruction module + DiT decoder $D$

[Text + Action + View] → DiT Encoder E
  → Spatial TB (M_spatial masking)
  → Temporal TB (M_time row-wise shift)
  → View TB (cross-view row-wise attn)
  → Token Reconstruction (F_s / F_t two-branch)
  → DiT Decoder D → Generative Loss + Reconstruction Loss

Reconstruction module is training-only; inference cost is identical to the baseline.

Key Design 1: Diffusion-related Mask Token

Bridges the gap between mask reconstruction targets and diffusion noise schedules:

$$m_\tau = (1 - \tau)\, f_n(\epsilon) + \tau\, p$$

High noise ($\tau$ large)

Learnable parameter $p$ dominates. Fine details are unknown, so $p$ estimates an average distribution to guide prediction direction.

Low noise ($\tau$ small)

Noise embedding $f_n(\epsilon)$ dominates. Encoding noise $\epsilon$ is explicitly available, enabling easier recovery of the original reconstruction target.

Source: Eq. 3 (p.5), ablated in Table 3a (p.8)

Key Design 2: Spatial and Temporal Masking

$\mathcal{M}_{spatial}$ (shared mask)

Same positions masked across all frames. Focuses on spatial context: scene objects, static structures. Reconstructed by spatial transformer $F_s$.

$\mathcal{M}_{time}$ (unshared mask)

Different positions masked per frame. Focuses on temporal context: object motions, dynamic changes. Reconstructed by temporal transformer $F_t$.

$$F = \begin{cases} F_s & \text{if } \mathcal{M} = \mathcal{M}_{spatial} \\ F_t & \text{if } \mathcal{M} = \mathcal{M}_{time} \end{cases}$$

Two-branch design forces context-appropriate reconstruction, outperforming shared structure (Table 4b: two-branch FVD 105.52 vs. shared FVD 121.37, p.8).

Key Design 3: Row-wise Shifted Temporal Self-Attention

Standard masked temporal attention requires 3D attention masks, incompatible with FlashAttention and expensive to compute.

Solution: For each row, mask the same number of tokens ($rW$ per row), drop invisible tokens, and directly connect visible tokens. This is equivalent to a shift augmentation: visible tokens from different frames align by row position, enabling context prediction without explicit attention masks.

  • Compatible with FlashAttention [6]
  • Training speed improved (Table 4a: row+shift att. reduces time from 0.368d to 0.329d per 10k steps, p.8)
  • Generation quality improved, especially at larger mask ratios

Strengths and Weaknesses

Strengths

  1. MR and diffusion complementarity is empirically validated across all settings
  2. Row-wise shifted attention is elegant: no attention masks, FlashAttention compatible, faster training
  3. Zero-shot Waymo transfer shows genuine generalization (33% FVD reduction)

Weaknesses

  1. No downstream task evaluation (planning, detection); "world model" claim not fully substantiated
  2. Performance gain partially from SD3 backbone choice, not just MR (acknowledged in Sec.9.2)
  3. Multi-view training limited to nuScenes only

Fundamental Limitations

  • No closed-loop evaluation: Open-loop video generation only; no planner integration or feedback mechanism
  • Autoregressive drift: FID reaches 37.7 at 10 seconds (Fig.7b, supplementary); quality still degrades over long horizons
  • SD3 dependence: Relative contribution of MR vs. backbone quality is difficult to disentangle
  • Controllability: Action module follows Vista design; challenging commands lead to unreasonable outputs (Sec.9.4)

Open Questions

  • Would MR benefit persist at 7B+ parameter scale?
  • Can the method be evaluated in closed-loop planning settings?
  • How sensitive is performance to text caption quality in OpenDV-2K?
